# 哈希表原理及应用
## 哈希表核心原理
+ 哈希表 + 数组 = ArrayHashMap、哈希表 + 双链表 = LinkedHashMap
+ **哈希表和Map不一样** -  根据底层数据结构不同，map的复杂度不一定都是O(1)
+ key为唯一，value可以重复
+ 哈希函数是把任意长度的非整数类型输入（key）→ 固定长度的整数输出（索引）
  + 这个函数的性能很关键，如果这个哈希函数复杂度是O(N)，那么哈希表的增删查改性能就会退化成O(N)
  + 这个函数还要保证的一点是，输入相同的key，输出也必须要相同，这样才能保证哈希表的正确性。
    > 示例 Java：全局唯一内存地址 负载因子：0.75
    > + int（[-2³¹, 2³¹ − 1]）可能为负数
    > + 用 % 来保证索引永远落在数组的合法范围
  
### 哈希冲突
0. 哈希桶 - 索引位置的存储单元
1. 纵向延伸 - 拉链法 O(k)为链表长度
2. 横向延伸 - 开放寻址法 O(k)为连续探查的次数

+ 负载因子 
  + 哈希表装满的程度，= size / table.length
  + 哈希表内元素达到负载因子时，哈希表会自动扩容。

+ 哈希表中键的遍历顺序是无序的
  + key 在底层 table 数组中的分布是随机的
  + 在扩容的时候索引发生变化
  
+ key必须为不可变类型
  + 复杂度退化到O(n)
  + 丢失会导致内存泄漏

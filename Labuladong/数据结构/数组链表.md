# 数组链表
## 数组
+ **「静态数组」就是一块连续的内存空间，我们可以通过索引来访问这块内存空间中的元素，这才是数组的原始形态。**
+ 动态数组 - 静态数组的基础上加上了API

### 静态数组
+ 数组名会自动衰退为指向首元素的指针（int*），所以数组首地址就是第一个元素的地址
+ 数组的元素类型决定了每个元素占用的存储空间大小
    + Byte 是内存与存储的基本寻址单位，也是大多数数据类型计算大小的基础。
    + 1 字节（Byte） = 8 比特（bit），bit 是最小的二进制单位，只能表示 0 或 1。
    + sizeof(int)（4字节）， sizeof(double)（8字节）
+ 数组的随机访问时间复杂度是O(1)

### **数据结构的职责就是增删查改**
+ 增
    1. 数组末尾追加（append）- O(1)
    2. 数组中间插入（insert）- O(n)
    3. 数组空间已满 - 连续内存必须一次性分配，分配完了之后就不能随意增减了 -> 扩容 O(n)
+ 删
    1. 删除末尾元素 - O(1)
    2. 删除中间元素 - O(n)
+ 查 
    1. 给出索引 - O(1)
    2. 遍历查找 - O(n)
+ 改 - O(1)

### 动态数组
+ **动态数组底层还是静态数组，只是自动帮我们进行数组空间的扩缩容，并把增删查改操作进行了封装，让我们使用起来更方便而已。**
+ 数组随机访问的超能力源于数组连续的内存空间，而连续的内存空间就不可避免地面对数据搬移和扩缩容的问题。

```cpp
vector<int> arr;
arr.push_back(i);
arr.insert(arr.begin() + 2, 666);
arr.pop_back();
arr.erase(arr.begin() + 2);
int index = find(arr.begin(), arr.end(), 666) - arr.begin();
```
``` python
arr = []
arr.append(i)
arr.insert(2, 666)
arr.pop()
arr.pop(2)
index = arr.index(666)
```

## 链表
+ 单链表节点只有一个next后节点指针；而双链表节点有两个指针，prev前节点指针，next后节点指针。

### 优缺点
+ 优点1 提高内存的利用效率
+ 优点2 不需要考虑扩缩容和数据搬移的问题
+ 缺点 链表不支持通过索引快速访问元素

### 增删查改（单/双）
+ 查/改 - O(n)
+ 增
  + 链表头部插入新元素 - O(1)
  + 链表尾部插入新元素 - O(n)
  + 链表中间插入新元素 - O(n)
+ 删
  + 链表中删除一个节点 - O(n)
  + 链表尾部删除元素 - O(n)
  + 链表头部删除元素 - O(1)

## 环形数组
**环形数组技巧利用求模（余数）运算，将普通数组变成逻辑上的环形数组，可以在O(1)的时间在数组头部增删元素。**
`i = (i + 1) % len(arr)`
+ 环形数组维护了两个指针，start 指向第一个有效元素的索引，end 指向最后一个有效元素的下一个位置索引。
+ 推荐开间设计 [start, end)

> *？ 为什么标准库中的动态数组容器不提供环形数组*
> 1. 两者设计目标不一致，环形数组无法扩容
> 2. 应用场景较窄

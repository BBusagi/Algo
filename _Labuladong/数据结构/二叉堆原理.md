# 二叉堆原理 (binary heap)
## 二叉堆核心原理及可视化
+ **二叉堆是一种能够动态排序的数据结构，是二叉树结构的延伸。**
  + 逻辑结构 - 完全二叉树
  + 具体实现 - 数组
+ 主要操作，sink（下沉）和 swim（上浮），维护二叉堆
+ 主要应用
  1. 数据结构优先级队列（Priority Queue）
  2. 排序方法堆排序（Heap Sort）
+ 二叉堆 vs BST
  + 二叉堆 局部有序，**找最值** O(1)
  + BST 全局有序，排序结构

### 🔒 二叉堆的性质
+ 二叉堆是一种动态排序的数据结构，即添加或删除元素之后会自动调整。
+ 二叉堆是优先级队列的底层实现，优先级队列能做到的，BST也能做到，但API和实现较为简单，能解决会优先选用
+ 任意节点的值，都必须大于等于（大顶堆）/小于等于（小顶堆）其左右子树所有节点的值
+ 二叉堆的左右子堆（子树）也是一个二叉堆

### 最常见的应用：优先级队列
+ API
  1. push 二叉堆堆顶插入元素 - O(logN)
  2. peek 返回堆顶元素 - O(1)
  3. pop 删除堆顶元素 - O(logN)
  4. 返回堆中元素的个数 - O(1)
+ 由于和FIFO的标准队列API比较像，所以被称之为队列，并且Heap只是最优实现之一

### 应用：堆排序
+ 乱序arr push到二叉堆中，再pop出得到一个有序的数组
+ 在思想上 近似优先级队列 时间复杂度 O(NlogN)，空间复杂度 O(N)
+ 在工程实现上 空间复杂度为O(1)，并把push和pop展开
+ 自动排序，但是增删元素的时间复杂度是O(logN)
  + push 堆顶插入一个元素，O(logN)
  + peek 返回堆顶元素，O(1)
  + pop 删除堆顶元素，O(logN)
  + size 返回元素个数，O(1)

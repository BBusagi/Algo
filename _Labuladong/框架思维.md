# 学习数据结构和算法的框架思维

### **总结一切数据结构和算法**
+ **数据结构的存储方式只有两种：数组（顺序存储）和链表（链式存储）**
+ 所有的数据结构的作用都是在于遍历和访问，增删查改等操作
+ 算法的本质为穷举，穷举的关键为无遗漏和无冗余
+ 算法框架可以做到无遗漏，利用信息可以做到无冗余
+ 在概率上要从上到下，在实现上要从下到上

### 数据结构的存储方式
+ 队列和栈：数组实现，扩容缩容的问题；链表实现，内存空间存储，节点指针
+ 图：邻接表是链表，邻接矩阵为二维数组。邻接表节省空间，邻接矩阵效率高
+ 哈希表：拉链法，链表特性，操作简单；线性法，数组特性，操作复杂
+ 树：数组实现就是堆，例如二叉堆;链表实现就是树，衍生设计二叉搜索树，AVL树，红黑树
+ 数组特性：连续存储，随机访问，索引快速访问，节约空间。但扩容需要重新分配并复制，时间复杂度O(N)；数组中间进行插入删除，时间复杂度O(N)
+ 链表特性：不存在扩容问题，指导前后指针的话，插入和删除为O(1)。无法随机访问；需要存储前后指针，消耗更多内存

### 数据结构的基本操作
+ 数据结构基本操作就是遍历 + 访问，具体来说就是：增删查改
+ 如何遍历 + 访问？两种形式：线性和非线性
+ 线性就是for/while迭代为代表，非线性就是递归为代表
  + 数组线性迭代遍历
  + 链表迭代/递归遍历
  + 二叉树非线性递归遍历
  + N叉树非线性递归遍历
+ 框架就是套路

### 算法的本质
+ **算法的本质就是穷举**
+ 例外有部分脑筋急转弯问题，密码学算法和机器学习算法
+ 「算法工程师」的「算法」，和「数据结构与算法」的「算法」完全不一样

### 穷举
+ 穷举的关键为**无遗漏**和**无冗余**
  1. 如何穷举？即无遗漏地穷举所有可能解。
  2. 如何聪明地穷举？即避免穷举过程中的冗余计算，消耗尽可能少的资源求出答案。
#### 「如何穷举」的难点问题
+ 递归类问题，例如回溯算法、动态规划系列算法的
+ 回溯算法是「遍历」的思维，而动态规划是「分解问题」的思维
+ 可以利用备忘录和空间压缩等技巧解决空间规划问题
#### 「如何聪明地穷举」的难点问题
+ 非递归算法问题，例如二分搜索就比for暴力循环更加高效
+ 节点连通可以用DFS/BFS，但是又UnionFind可以将复杂度降到O(1)
+ 贪心算法在条件合适的情况下比动态规划更加高效

### 数组/单链表系列算法
+ 单链表双指针 「如何聪明地穷举」
+ 数组双指针 「如何聪明地穷举」
+ 二分搜索
+ 滑动窗口，快慢双指针
+ 前缀和、差分数组，通过额外维护一个数组可以避免不必要的循环
